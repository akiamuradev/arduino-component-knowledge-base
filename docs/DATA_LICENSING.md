## Source snapshot карточки

При создании draft backend сохраняет неизменяемый snapshot происхождения:

- название источника, repository и original URL;
- полный resolved commit SHA и, если применимо, запрошенный tag;
- путь исходного файла и имя entry/symbol;
- имя и версия parser;
- название, SPDX identifier и URL лицензии;
- attribution и описание выполненных преобразований;
- время импорта и provenance нормализованных полей.

PostgreSQL хранит эти metadata. Binary media хранится только в private MinIO и не включается в
repository import автоматически. При публикации source snapshot копируется в immutable revision,
чтобы последующие изменения registry не переписали историю уже опубликованной карточки.

## Редакционный процесс

Repository parser создаёт только `draft`. До публикации teacher или administrator проверяет
содержимое карточки, а backend повторно проверяет наличие revision, origin, license, attribution и
modifications notice. Frontend показывает эти сведения read-only; источник истины для проверки —
backend. Ручная карточка без стороннего материала может не иметь source snapshot.

Если одна карточка объединяет несколько допустимых источников, каждый snapshot сохраняется и
отображается отдельно. Решение о duplicate merge подтверждает только administrator; merge не
должен удалять source snapshots или смешивать их лицензии в одну вымышленную лицензию.

## Изменение лицензии, отзыв и спор

Новая лицензия upstream не применяется задним числом автоматически. Для новой revision создаётся
новый snapshot после повторной проверки условий. Если источник отозван, право использования
оспаривается или attribution оказался неверным, administrator должен:

1. отключить источник для новых jobs;
2. приостановить публикацию затронутых drafts;
3. определить затронутые карточки по source/revision/file/entry;
4. архивировать или исправить карточки по результату проверки;
5. сохранить audit trail решения и не удалять данные без отдельной обоснованной процедуры.

Приложение не делает юридический вывод о совместимости лицензий и не подтверждает права оператора
на конкретное использование. Ответственность за проверку условий и необходимой attribution перед
публикацией остаётся у оператора системы.

## Поддерживаемые источники

Новый импорт разрешён только из Seeed Studio Wiki (`GPL-3.0-only`) и Official KiCad Symbols
(`CC-BY-SA-4.0`). Arduino-Tex и Portal-PK неактивны. AlexGyver отключён с причиной
`owner_denied_usage`; материалы этого источника нельзя импортировать.
